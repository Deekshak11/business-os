# Offer Enhancers

## When to use

- “I’ll think about it” epidemic  
- Stack is strong but not compelling in presentation  
- Need risk reversal  
- Naming is generic (“coaching package”)  

## Enhancer stack

### 1. Bonuses
- Solve **secondary obstacles** to the dream outcome.  
- High perceived value, controlled delivery cost.  
- Name bonuses; price-anchor each bonus’s value.  
- Order: core offer → bonuses that remove friction → tools/templates → community/access.

### 2. Guarantees (risk reversal)
- Increase **perceived likelihood** (Value Equation).  
- Types (conceptual): unconditional, conditional, anti-guarantee (strong stance), performance/outcome-based, stacked guarantees.  
- Match guarantee to what the market fears most.  
- Only guarantee what fulfillment can support.

### 3. Scarcity (honest)
- Limited seats, cohorts, bonuses, or capacity.  
- **Real** scarcity only — fake scarcity destroys trust.  
- Capacity-based scarcity is natural in services.

### 4. Urgency (honest)
- Deadlines, cohort starts, expiring bonuses, seasonal windows.  
- **Real** urgency only.  
- Urgency without reason is ignored by sophisticated buyers.

### 5. Naming
- Specific > vague.  
- Outcome + avatar + mechanism beats “premium package.”  
- Makes the offer memorable and harder to commodity-compare.

## Operator checklist

- [ ] Each bonus maps to a real objection or sub-problem  
- [ ] Guarantee is clear, credible, and operationally safe  
- [ ] Scarcity/urgency reasons are true and stated  
- [ ] Name is unique and outcome-linked  
- [ ] Presentation order maximizes value before price
