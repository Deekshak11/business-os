# **SECTION 0 — PREREQUISITES & HOW TO USE**

---

### **Prerequisites:**

**→ REQUIREMENT: Block I complete** (Docs 1-4)

**Especially critical:**

✅ **Doc 4** (Offer engineered) — the blueprints here PRESENT an offer (they don't create it — the offer already exists from Doc 4)

✅ **Doc 3** (Big Idea, Voice) — the copy EXPRESSES positioning (the Big Idea permeates the structure, the Voice = tone)

✅ **Doc 2** (State/Stage) — DETERMINES which blueprint to choose (the Decision Tree in Section 1.7 uses State/Stage as input)

---

**Optional but useful:**

Doc 1 (Laws) — understand WHY the structures work (Axiom 1: copy = salesmanship, sales structure = copy structure).

---

### **Time:**

**Complete read-through:** 4-5 hours (Part I principles + Part II 8 blueprints).

**Practical use:** 30min-2h (choose format with Decision Tree → go directly to the specific blueprint → follow the structure → write the copy).

---

### **How to Use This Document:**

**FIRST TIME (Copywriter Learning):**

1.  Part I completely (universal principles — the foundation for all formats)
2.  Part II: Read ALL 8 blueprints (understand the options, when to use each)
3.  Decision Tree (learn how to choose)
4.  Practice: Choose 1 blueprint, write copy following it

---

**PRACTICAL USE (Copywriter Executing a Project):**

1.  **Decision Tree** (Section 1.7 — 5 min): Inputs (State/Stage from Doc 2, Price, Goal) → Output (recommended format)
2.  **Specific Blueprint** (Section 2.X.Y — 15-30 min read): Structure, components, sequence
3.  **Checklist** (Part III — 10 min): Validate during/after writing
4.  **Write** (following the blueprint — 2-8h depending on format)

**Total:** 30 min prep + 2-8h execution = complete copy in the appropriate format.

---

### **Output:**

**Complete, ready-to-use copy:**
- Sales Letter (6-10k words)
- VSL Script (30-45min, 5-8k words)
- Webinar (60-90min outline + slides)
- Email (150-400 words)
- Ad (15-90 second script)
- Landing page (300-800 words)
- Etc.

---

### **Connection to Other Docs:**

**Doc 5 (this one)** = STRUCTURES (blueprints, scaffolding, architecture).

**Docs 6-7** = COMPONENTS (headlines, bullets — fill the blueprint sections).

**Doc 8** = PROOF (testimonials, data — the proof section of blueprints).

**Doc 9** = LANGUAGE (words, tone — polish the copy).

**Docs 10-13** = SYSTEMS (funnels, sequences, optimization, swipes).

---

**Workflow:**

Doc 5 (choose structure) → Doc 6 (write headline) → Doc 7 (bullets) → Doc 8 (proof) → Doc 9 (polish language) → Launch.

**Structure first** (foundation), components next (fill), polish last.
