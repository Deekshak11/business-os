## **SECTION 3.3 — THE REPRESENTATIVENESS HEURISTIC**

---

### **THE PRINCIPLE:**

> "We judge the probability that something belongs to a category by how well it fits the stereotype of that category."

---

**An S1 shortcut:** "If A looks like B (it represents the stereotype of B) → A is probably B."

It ignores **base rates** (general frequencies).

---

### **THE EXPERIMENT (The Linda Problem):**

> "Linda is 31 years old, single, outspoken, and very bright. She majored in philosophy. As a student, she was deeply concerned with issues of discrimination and social justice, and also participated in anti-nuclear demonstrations.
>
> Which is more probable?
> A) Linda is a bank teller
> B) Linda is a bank teller AND is active in the feminist movement"

---

**85% answered:** **B**

---

**A LOGICAL ERROR (The Conjunction Fallacy):**

B is a **subset** of A (a bank teller AND a feminist ⊂ a bank teller).

Logically: P(A and B) ≤ P(A) **always** (the probability of a conjunction cannot exceed the probability of a component).

Therefore: B **cannot** be more probable than A.

---

**The correct answer:** A is more probable (or equal, never less).

---

**Why did 85% get it wrong?**

**System 1 (Representativeness):**

Linda's description (philosophy, social justice, demonstrations) **perfectly represents** the stereotype of an active feminist.

"Linda looks like a feminist" → Therefore: B (a bank teller + a feminist) seems more "Linda-like" than just A (a bank teller).

**Representativeness > the logic of probability** (S1 ignores the conjunction rule).

---

**System 2** (the logical one) could have corrected it ("B ⊂ A, therefore A ≥ B").

**BUT:** 85% did not activate S2 (or they activated it but it failed to override S1 — representativeness was too strong).

---

Kahneman:

> *"When people judge probability by representativeness, they ignore base rates and focus entirely on the description.*
>
> *System 1 does not understand the logic of probability. It understands similarity/pattern matching."*

---

### **APPLICATIONS:**

```
┌─ TACTIC #1: SPECIFIC PERSONAS (Even Improbable Ones) ───┐
│                                                         │
│ In headlines, bullets, and stories:                    │
│                                                         │
│ Use ULTRA-SPECIFIC personas (even if they are statistically rare/improbable).│
│                                                         │
│ Bencivenga (a classic bullet):                         │
│ "How a shy, introverted accountant became a millionaire │
│ in 18 months..."                                       │
│                                                         │
│ Statistically: Shy accountants rarely become         │
│ millionaires quickly (it's improbable).                │
│                                                         │
│ BUT: The story **represents** a dramatic transformation│
│ (shy → rich = maximum contrast).                       │
│                                                         │
│ System 1 (Representativeness): "If HE (shy, improbable)│
│ could do it, I (less shy? more capable?) can do it too."│
│                                                         │
│ Representativeness outperforms real probability.       │
│ An underdog story is more persuasive than a "common person" story.│
│                                                         │
│ Other examples:                                         │
│ • "A 67-year-old grandmother with arthritis..." (age + │
│   a physical limitation = improbable → powerful)       │
│ • "A high school dropout..." (a low education → success = │
│   a contrast)                                          │
│ • "A broke single mom..." (a difficult situation → a   │
│   dramatic transformation)                             │
│                                                         │
│ → REQUIREMENT: Doc 7 (Bullets — the Underdog stories type)│
└─────────────────────────────────────────────────────────┘

┌─ TACTIC #2: POSITIVE STEREOTYPES (When They Help) ─────┐
│                                                         │
│ Use cultural stereotypes (S1 loves them) when they are favorable:│
│                                                         │
│ • "A German doctor designed..." (Germans = precision,  │
│   engineering — a positive stereotype)                 │
│                                                         │
│ • "A French chef approved..." (the French = superior cuisine│
│   — a stereotype)                                      │
│                                                         │
│ • "A Swiss engineer developed..." (the Swiss = quality,│
│   watches — a stereotype)                              │
│                                                         │
│ S1 uses representativeness (German = precision) → The product│
│ seems precise/reliable (an automatic association).     │
│                                                         │
│ It works EVEN if the stereotype isn't 100% true        │
│ (S1 doesn't analyze — it accepts the pattern).         │
│                                                         │
│ ETHICAL: Do not use NEGATIVE or offensive stereotypes │
│ (racism, sexism — it's morally wrong + a legal backfire).│
└─────────────────────────────────────────────────────────┘

┌─ TACTIC #3: AVOID NEGATIVE STATISTICS WITHOUT A REFRAME ──┐
│                                                         │
│ ❌ WRONG (A negative stat, without a reframe):         │
│ "95% of new entrepreneurs fail in the first 5 years."  │
│                                                         │
│ System 1 (Representativeness): "I = a new entrepreneur│
│ → the 95% failure rate applies to ME → I'm going to fail."│
│ [An application via representativeness — it's demotivating]│
│                                                         │
│ ✅ CORRECT (A Reframe — segment it):                   │
│ "95% of entrepreneurs WITHOUT A SYSTEM fail in 5 years.│
│                                                         │
│ BUT: 78% of those who USE our system survive for 5+ years│
│ (and 64% grow 3x+).                                    │
│                                                         │
│ The difference? A SYSTEM (not talent, luck, or timing)."│
│                                                         │
│ System 1: "I = an entrepreneur WITH a system (not 'without a system'). Therefore: the 78% success rate (not the 95% failure).│
│ I represent the SUCCESS category (not the failure category)."│
│                                                         │
│ A reframe via segmentation (with vs. without) = representativeness applied positively.│
└─────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════
SUMMARY — REPRESENTATIVENESS
• The principle: Judgment via a fit with a stereotype (not base rates)
• The experiment: The Linda Problem (85% were logically wrong — the conjunction fallacy)
• The mechanism: S1 pattern matching (it ignores probabilities)
• Applications:
  1. Specific personas (underdog stories — Bencivenga)
  2. Positive stereotypes (German = precision)
  3. Reframe negative stats (segment: with vs. without a system)
• Power: ⭐⭐⭐ (Identification via representation)
```
